Understanding Cybersecurity

Cybersecurity is the practice of protecting systems, networks, and programs from digital attacks. These attacks are usually aimed at accessing, changing, or destroying sensitive information, extorting money, or interrupting normal business processes.

Core Security Concepts

Cryptographic Hashing

A hash function is a one-way mathematical algorithm that converts input data into a fixed-size string of characters. The same input always produces the same hash, but any change to the input produces a completely different hash.

Key Properties:

  • Deterministic: Same input = same output
  • One-way: Cannot reverse to find original
  • Collision-resistant: Hard to find two inputs with same hash
  • Avalanche effect: Small change = completely different hash

Common Algorithms:

  • SHA-256: Industry standard, 256-bit output
  • SHA-3: Latest SHA family member
  • MD5: Deprecated, still seen in legacy systems

File Integrity Monitoring

File Integrity Monitoring (FIM) is a security practice that validates the integrity of operating system and application files by comparing their current state against a known, trusted baseline.

Use Cases:

  • Detecting malware infections
  • Identifying unauthorized changes
  • Compliance requirements (PCI DSS, HIPAA)
  • Digital forensics investigations

Real-World Tools:

  • OSSEC (Open Source)
  • Tripwire
  • AIDE (Advanced Intrusion Detection)

Phishing Attacks

Phishing is a type of social engineering attack where attackers disguise themselves as trustworthy entities to trick victims into revealing sensitive information or installing malware.

Types of Phishing:

  • Spear Phishing: Targeted attacks on specific individuals
  • Whaling: Targeting high-profile executives
  • Smishing: Phishing via SMS
  • Vishing: Voice phishing via phone calls

Red Flags:

  • Urgent or threatening language
  • Requests for sensitive information
  • Suspicious sender addresses
  • Mismatched or suspicious URLs

Password Security

Passwords are the first line of defense for most accounts. Strong password practices are essential to prevent unauthorized access through brute force or dictionary attacks.

Attack Methods:

  • Brute Force: Trying all combinations
  • Dictionary Attack: Using common words/passwords
  • Rainbow Tables: Pre-computed hash lookups
  • Credential Stuffing: Using leaked credentials

Best Practices:

  • Use 12+ characters minimum
  • Mix character types
  • Use unique passwords per account
  • Enable two-factor authentication

Digital Forensics

Digital forensics involves the recovery, investigation, and analysis of material found in digital devices, often in relation to computer crime or cybersecurity incidents.

Process Steps:

  • Identification: Recognize potential evidence
  • Preservation: Secure and document evidence
  • Analysis: Examine data systematically
  • Documentation: Record findings thoroughly
  • Presentation: Report in understandable terms

SOC Operations

A Security Operations Center (SOC) is a centralized unit that deals with security issues on an organizational and technical level. SOC analysts monitor, detect, and respond to cybersecurity threats.

SOC Analyst Tiers:

  • Tier 1: Alert triage and initial investigation
  • Tier 2: Deep investigation and response
  • Tier 3: Threat hunting and advanced analysis

Key Tools:

  • SIEM (Security Information Event Management)
  • EDR (Endpoint Detection and Response)
  • Threat Intelligence Platforms
